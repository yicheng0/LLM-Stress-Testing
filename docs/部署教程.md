# Docker 部署教程

这是这套项目的推荐部署方式：`Docker Compose` 单机部署。仓库已提供：

- `docker-compose.yml`
- `Dockerfile.backend`
- `Dockerfile.web`
- `nginx.conf`
- `scripts/docker-self-update.sh`

默认端口已避开常见端口：

- Web 面板：`18080`
- 后端直连调试：`18081`

## 1. 前置条件

- 已安装 Docker
- 已安装 Docker Compose
- 服务器可访问目标模型 API

建议目录：

- 代码：`/opt/model_rpm_test`
- 数据：`/opt/model_rpm_test/data`
- 结果：`/opt/model_rpm_test/results`

## 2. 准备配置

先复制环境变量模板：

```powershell
copy .env.example .env
```

常用配置：

- `DATABASE_URL=sqlite:///./data/llm_test.db`
- `RESULTS_DIR=./results`
- `MAX_RUNNING_TESTS=2`
- `MAX_CONCURRENCY_PER_TEST=500`
- `WEB_PORT=18080`：Web 面板宿主机端口，可改成其它不冲突端口
- `BACKEND_PORT=18081`：后端宿主机调试端口，可改成其它不冲突端口
- `SELF_UPDATE_ENABLED=true`：启用面板里的版本检测和更新按钮
- `SELF_UPDATE_COMMAND=sh /app/scripts/docker-self-update.sh`：Docker 部署下推荐使用的面板更新命令
- `SELF_UPDATE_TIMEOUT_SEC=900`：单次更新命令超时

如果前端和后端同域名部署，通常不需要额外设置 `VITE_API_BASE_URL` 和 `VITE_WS_BASE_URL`。

## 3. 一键启动

启动命令：

```powershell
docker compose up -d --build
```

启动后访问：

- Web 面板：`http://服务器IP:18080`
- 后端健康检查：`http://服务器IP:18081/api/health`
- 也可以通过 Web 端口检查：`http://服务器IP:18080/api/health`

如果端口冲突，直接改 `.env`：

```env
WEB_PORT=19080
BACKEND_PORT=19081
```

然后重启：

```powershell
docker compose up -d
```

## 4. 日常更新

你前面提到的“检查版本 / 点一下更新”，在 Docker 里可以做成很简单的流程：

1. 拉取新代码
2. `docker compose up -d --build`
3. 自动替换旧容器

手工更新命令：

```powershell
git pull
docker compose up -d --build
```

如果希望面板里的“检测版本 / 更新”按钮可用，`.env` 里这样配置：

```env
SELF_UPDATE_ENABLED=true
SELF_UPDATE_COMMAND=sh /app/scripts/docker-self-update.sh
```

当前 Docker 配置会把项目目录挂载进后端容器，并挂载 Docker Socket。面板点击更新后，后端会执行：

```text
git pull --ff-only
docker compose up -d --build
```

更新过程中服务可能短暂中断，浏览器刷新后即可进入新版本。

注意：挂载 Docker Socket 后，后端容器具备重建本项目容器的能力。这个功能只建议部署在可信内网，不要把面板直接暴露到公网。

## 5. 数据保留

以下目录要挂载到宿主机，不要放在容器内部：

- `data/`
- `results/`

这样即使容器重建，SQLite 状态和测试结果也不会丢。

## 6. 常用命令

查看日志：

```powershell
docker compose logs -f
```

重启：

```powershell
docker compose restart
```

停止：

```powershell
docker compose down
```

## 7. 自检

服务起来后执行：

```powershell
.\scripts\smoke_check.ps1
```

它会检查后端健康、任务列表接口和前端是否可访问。

## 8. 最简建议

如果你只想快点上线，就按这个顺序做：

```text
准备 .env -> docker compose up -d --build -> 浏览器访问 -> smoke_check.ps1
```
