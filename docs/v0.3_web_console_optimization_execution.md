# v0.3 Web Console Optimization Execution Notes

## Scope

This document covers local startup, environment variables, lightweight SQLite state, API key handling, result retention, cleanup, and smoke validation for the v0.3 web-console optimization work.

## Local Startup

Backend:

```powershell
cd D:\model_rpm_test
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r backend\requirements.txt
python -m uvicorn backend.app.main:app --host 127.0.0.1 --port 8000 --reload
```

Frontend:

```powershell
cd D:\model_rpm_test\frontend
npm install
npm run dev
```

Open `http://127.0.0.1:5173`. The Vite dev server proxies `/api` and `/ws` to `http://127.0.0.1:8000`, so local development does not require frontend API environment variables.

## Environment Variables

Backend variables:

| Variable | Default | Purpose |
| --- | --- | --- |
| `DATABASE_URL` | `sqlite:///./data/llm_test.db` | SQLAlchemy database URL for task state. |
| `RESULTS_DIR` | `./results` | Directory for summaries, details JSONL, reports, and matrix CSV output. |
| `MAX_RUNNING_TESTS` | `2` | Maximum concurrently running web-console tasks. |
| `MAX_CONCURRENCY_PER_TEST` | `500` | Per-task concurrency guardrail. |

Frontend variables:

| Variable | Default | Purpose |
| --- | --- | --- |
| `VITE_API_BASE_URL` | empty | Optional API base URL outside local proxy mode. |
| `VITE_WS_BASE_URL` | empty | Optional websocket base URL outside local proxy mode. |

Use `.env.example` as the documented template. Do not add real credentials to the example file.

## SQLite State Store

The web console defaults to SQLite at `data/llm_test.db`. The database is intentionally lightweight and local-first. It stores:

- task metadata and current status
- sanitized task configuration
- event log entries
- summary JSON and generated report file paths

The report payloads and detailed per-request JSONL files remain in `RESULTS_DIR`, not directly embedded in the SQLite database.

## API Key Handling

API keys are supplied per test run and are required to execute a load test. The backend passes the key to the in-memory runner configuration for the active task, but removes `api_key` before saving task configuration to SQLite. Generated matrix summaries also remove `api_key` from their saved config.

Operational rules:

- Do not persist API keys in `.env`, `.env.example`, README examples, screenshots, logs, or result artifacts.
- Prefer shell environment variables for CLI usage, for example `$env:API_KEY` in PowerShell.
- Treat existing `results/` artifacts as sensitive until reviewed.

## Result Retention

Generated results are local operational artifacts and should be considered short-term by default. The default result locations are:

- CLI runs: files directly under `results/`
- Web-console runs: files under `results/<task_id>/`

Keep only the files needed for current investigation, comparison, or PR evidence. Export or archive selected reports before cleanup if they are needed longer-term.

## Cleanup Strategy

Preferred cleanup path:

1. Use the web UI delete action for a task. This removes the task rows and its task-specific `results/<task_id>/` directory.
2. For old CLI-generated files directly under `results/`, delete only files you have reviewed and no longer need.
3. For a full local reset, stop backend/frontend processes, then remove `data/llm_test.db` and selected `results/` contents.

Do not run broad cleanup while other tests are running. Do not remove another developer's result files unless that is explicitly coordinated.

## Smoke Test

Start backend and frontend, then run:

```powershell
cd D:\model_rpm_test
.\scripts\smoke_check.ps1
```

The script checks:

- required backend/frontend files exist
- `data/` and `results/` directories are present or clearly reported
- backend `GET /api/health` returns `status: ok`
- backend `GET /api/tests?page=1&page_size=1` responds
- frontend root page is reachable

The smoke test is read-only from the application perspective: it does not create a test task, does not call a model API, and does not modify SQLite state.

Useful variants:

```powershell
.\scripts\smoke_check.ps1 -BackendUrl http://127.0.0.1:8000 -FrontendUrl http://127.0.0.1:5173
.\scripts\smoke_check.ps1 -SkipFrontend
```
